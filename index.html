JDBC连接数据库

发表于 2019-08-04

访问数据库流程

  1. 驱动管理器————加载JDBC驱动程序

  2. 连接数据库————建立与数据库的连接

  3. SQL语句————发送SQL语句

  4. 结果集————得到查询结果

常用接口

  • Driver接口

    • 装载MySQL驱动

      Class.forName(“com.mysql.jdbc.Driver”)

    • 装载Oracle驱动

      Class.forName(“oracle.jdbc.driver.OracleDriver”)

  • DriverManager接口

    • DriverManager是JDBC的管理层，作用于用户和驱动程序之间
    • DriverManager跟踪可用的驱动程序，并在数据库和相应的驱动程序之间建立联系

  • Connection接口

    • Connection与特定数据库的连接（会话），在连接上下文执行SQL语句并返回结果

    • DriverManager的getConnection()方法建立在JDBC URL中定义的数据库Connection连接上

    • 连接Mysql数据库：

    • 连接Oracle数据库

      C

  • Statement接口

    • 用于执行静态SQL语句并返回它所生成结果的对象

    • 三种Statement类

      1. Statement

        由createStatement创建，用于发送简单的SQL语句（不带参数的）

      2. PreparedStatement

        继承自Statement，由prepareStatement创建，用于发送含有一个或多个输入参数的sql语句。PreparedStatement对象比Statement对象的效率更高，并且可以防止SQL注入。我们一般都用PreparedStatement。

      3. CallablesStatement

        继承自PreparedStatement。由方法prePareCall创建，用于调用存储过程。

    • 常用的Statement方法

      • excute()：运行语句，返回是否有结果集
      • excuteQuery()：运行select语句，返回ResultSet结果集
      • excuteUpdate()：运行insert/update/delete操作，返回更新的行数

报错JVM_Bind

发表于 2019-01-02

运行Eclipse项目时，报如下错误

Address already in use: JVM_Bind <null>:8080

问题所在：8080端口被占用了，把8080端口的进程kill掉就ok了
解决办法如下：
  1. 先查看所有端口号：netstat -ano 该命令列出所有端口号
    找到8080端口，它的PID为1376

    1. 打开任务管理器，查看详细信息，把PID为1376的这个javaw.exe右键结束进程就可以了，一般都是这个占用8080端口

安装node.js

npm:它是node包管理和分发工具

  • init：工程初始化命令

  • npm install 模块名：用于安装某一个模块（如express（node的web框架））

  • npm install -g：-g代表全局安装

  • npm root -g：可以查看全局目录在哪

  • npm install：一键下载工程所需要的模块

  • npm run dev：运行某一个工程

  • npm run build：编译工程，生成之后会多出一个dist文件夹，里面只有一个静态页面和一个static文件夹

    这种工程我们称之为单页Web应用（single page application,SPA），就是只有一张Web页面的应用，是加载单个HTML页面并在用户与应用程序交互时动态更新改页面的Web应用程

webpack

  • webpack官网：https://www.webpackjs.com/

  • webpack是一个前端资源加载/打包工具。它将根据模块的依赖关系进行静态分析，然后将这些模块按照指定的规划生成对应的静态资源。

  • 安装webpack和webpack-cli：npm install webpack webpack-cli –save-dev

  • 把js、css、图片等打包成一个js文件，然后一个静态页面

第一章 初识Vue.js

  • 一个渐进式的JavaScript框架

  • MVVM模式（Model-View-ViewModel）：当View（视图层）改变时，会自动更新到ViewModel（视图模型），反之亦然。View和ViewModel之间通过双向绑定（data-binding）建立联系

  • 可以直接通过加载script加载CDN文件

第2章 数据绑定和第一个Vue应用

  • 创建一个实例，通过构造函数Vue就可以创建一个Vue的根实例，并启动Vue应用
    变量app就代表了这个Vue实例。事实上，几乎所有的代码都是一个对象，写入Vue实例的选项内的。
    首先，必不可少的一个选项就是el。el用于指定一个页面中已存在的DOM元素来挂载Vue实例，它可以是HTMLElement，也可以是CSS选择器，比如：

  • Vue的生命周期（常用的有如下）

    1. created 实例创建完成之后调用，此阶段完成了数据的观测等，但尚未挂载，$el还不可用。
